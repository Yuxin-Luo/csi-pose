## 1. 调研/开发目标

为组内汇报（受众：老师 + 领导）做一个**浅色调、单文件 HTML**，整合 csi-pose 开源项目复现工作的：
- **工作流程**（7 天 / 7 个阶段）
- **中途遇到的关键 bug + 解决方式**
- **最终达到的效果**（客观指标 + 演示 demo 视频）

PPT 15-20 页折算为 HTML 的 18 页。

## 3. 关键发现 / 决策依据

### 3.1 受众与深度（用户决策）
- **老师 + 领导** → 不深入技术细节，但需要用**简单语言**讲清"项目用了什么技术、怎样同步"
- 参考 `docs/csi-pose-techstack.html` 的简明版式（少公式、多卡片、多 callout）

### 3.2 内容权重（用户决策）
- **过程 + bug 为主**（8 页 ≈ 44%）
- **最终效果精简为 2-3 页**
- 技术背景 3 页（用户已了解基础概念时一带而过）
- 收尾 1 页（数据溯源）

### 3.3 bug 组织方式（用户决策）
- **按主题分类**（固件层 / 物理层 / 训练层 / 推理层 / 流程编排）
- 比"按时间阶段"更突出问题导向，受众更容易抓住"踩了什么坑"

### 3.4 视频处理（用户决策）
- HTML 内仅放**文字描述 + 超链接**，点击 → 系统播放器打开 mp4
- 录屏：`runs/录屏 2026年07月12日 18时57分22秒.mp4`（73.3 秒）

### 3.5 客观指标（用户决策）
- **一张表**呈现核心数字（不堆数据）
- 字段：链路 loss / cam fps / RT 推理延迟 / PCK@0.5 / 端到端 pipeline 状态

### 3.6 成功标准（用户决策）
- "演示 demo 视频本身能跑通 esp32 → 人体姿态估计画火柴人系统"
- HTML 内放超链接，**视觉上证明**而非仅靠数字

### 3.7 诚实声明（继承 CLAUDE.md §1.1）
- 跌倒阈值单会话标定 → 明确承认
- amp-only 模型 PCK@0.5=55.5% vs norm PCK@0.5=78.9% 但不能用 → 明确差距
- 写入"诚实声明"页，让受众看到严谨态度

## 4. 18 页大纲（最终版）

### A · 项目背景（3 页）
1. **封面** — 标题、一句话目标、日期、署名
2. **目录 / sticky nav** — 8 个一级章节锚点
3. **项目是什么 + 总体路线图** — 一句话 + 5 子系统卡片（subsystem-grid）

### B · 用了什么技术（3 页，简明语言）
4. **硬件拓扑** — 6 块 ESP32-S3 = 3×3 link matrix，内嵌 SVG 示意图
5. **时钟同步** — 为什么不做"显式同步"：TX beacon 同时性 + 每板下包络拟合兜底
6. **CSI 数据 → 教师打标签 → 训练 → 实时推理** — 一页串完（一图 + 四步骤文字）

### C · 过程 + Bug（8 页，主体，bug 按主题分类）
7. **7 天时间线总览** — 阶段 1-7 一张总览
8. **固件层 Bug** — 4 层 bug 链：CONFIG_ESP_WIFI_CSI_ENABLED / sdkconfig 不被 defaults 重读 / console 走 UART0 / csi_link 走 UART0
9. **物理层 Bug** — 95% loss 误判修正 + USB autosuspend + 桥接 startup accounting 伪影
10. **训练 / 数据 Bug** — rawlog chunked streaming 误读 + OOM batch=64 + onnxruntime CPU-only + cam fps 物理上限
11. **推理层 Bug** — 骨架坍缩对角线（in_ch 不匹配）+ RTMDet load timeout
12. **流程编排 Bug** — boot 6 次失败 silent death + `input()` EOFError
13. **方法论小结** — 5 报错就停、不打补丁、根因优先、决策可追溯
14. **23 份 dev_doc 留底** — 工作量事实呈现

### D · 效果展示（3 页，精简）
15. **客观指标表格** — 一张表：链路 loss / cam fps / RT 推理延迟 / PCK@0.5
16. **演示 Demo 视频** — 录屏 73.3 秒**超链接** + 一句话描述
17. **诚实声明 + 下一步计划** — 合并为一页，体现严谨

### E · 收尾（1 页）
18. **数据溯源** — dev_doc 23 份索引 + runs/ 产物清单 + 论文链接
