1系统全景

一句话：把 WiFi CSI（信道状态信息）的振幅张量当成"身体的无线电影子"， 训练一个神经网络从中回归出 18 关节 2D 坐标；再在姿态之上叠加规则式状态机做跌倒检测。

csi-pose 把 Intel 5300 上的 WiSPPN（3×3 天线矩阵）移植到 6 块 ESP32-S3 上（3TX×3RX = 9 链路， 链路矩阵）。摄像头只在采集阶段用——RTMDet+RTMPose 当教师打伪标签， 训练完成后推断只需 CSI，不再需要摄像头。

2硬件拓扑（浅）

2.1 3TX × 3RX 链路矩阵

6 块 ESP32-S3 物理上分离，每块都是独立的 NIC。3 个 TX + 3 个 RX 形成 9 条无线链路， 即 3×3 link matrix（不是单 NIC 的 antenna matrix）。

3时钟同步（中）

板子的 esp_timer 彼此独立、也和主机独立。每包到达都用单一主机时钟 time.time_ns() 打戳， 但 USB 串口延迟会让包迟到。关键观察：USB 延迟只能让包迟到、不能早到。 因此 (t_esp, t_host) 散点的下包络（lower convex hull）就是时钟变换的无偏估计。

3.1 时钟模型

t_host = (1 + ρ) · t_esp + β + δ_usb + ε,    δ_usb ≥ 0

其中 ρ 是晶振频率偏差（ppm 级），β 是常数偏移，δ_usb 是 USB 串口批处理延迟（非负）。 取下包络等于假设 δ_usb = 0。

3.2 拟合策略

按 window_s = 600 s 分段做最小二乘（窗口中心 centers_s），窗口间线性插值吸收温度漂移：

  1. 按 1 s 桶取每个桶内最小延迟（抑制单侧抖动）
  2. 对桶代表点求下凸包（monotone chain 下沿）
  3. 每个 600 s 窗口内做一次 1 阶多项式拟合（α, β）
  4. 窗口中心间线性插值（外推到首尾窗口）
  5. 按 boot_id 切分 epoch（重启 = 新时钟）

3.4 视频-视频配对修正

摄像头帧和 CSI 流的物理时间不对齐（摄像头有软件抓帧延迟），用测得的系统偏差补偿：

anchor' = t_vid − (Δ_cam − Δ_csi) = t_vid − 156.12 ms

"156.12 ms" 是 STOP 事件 + 监视器翻拍实测出来的常数（写入 configs/pairing.json）。 修正在窗口选择阶段做，但 /samples/t_ns 始终保留原始视频戳，便于后续 audit。

💡 ±15 ms 的现实意义：跌倒瞬间人体速度 2-3 m/s，对应 3-5 cm 的标签噪声。 这就是为什么训练要加 confidence gating > 0.2（loss.py:C_FLOOR）和分布平均。

💡 为什么要保留空房帧？空房是负样本——CSI 振幅会因环境微小变化（空气流动、温漂）有微小波动， 模型必须学会把这种"无人但有扰动"的情况输出低置信度（ĉ → 0），而不是乱猜姿态。 损失函数中 w = 𝟙[presence] · max(c_gt, 0.2) 自然实现了这一点：负样本的 c_gt = 0， ĉ 被训练目标拉向 0，姿态坐标项则自动屏蔽。

4.4 QA gate

训练前随机抽 < 2% 帧人工复核，确保 RTMDet/RTMPose 在该房间、该视角下的失败率可接受。 失败来源主要是遮挡、姿态极端（小尺寸躯干）、低光照。代码路径：teacher/csi_teacher/qa.py。

5特征工程 ★（深）

本节是项目的核心创新点之一：从原始 CSI 帧到训练张量的所有变换都在这里。 代码主入口 train/csi_train/data.py，配合 host/csi_pipe/samples.py（构建窗口）。

5.1 输入张量构造

每条 CSI 帧给出 1 个 (K=56,) 子载波振幅向量。沿时间轴连续取 5 包（50 ms），展平得到：

X[c, i, j] = A^(i,j)_{t−(4−p)·Δ, k},    c = p·56 + k,   p = 0..4,   Δ = 10 ms

其中 i = rx_i（接收板索引 0..2）、j = tx_j（发送板索引 0..2）。 最终形状：X ∈ ℝ^(280×3×3)，280 = 5 包 × 56 子载波。

5.2 三个可选特征

特征选择由 configs/train.yaml:features 决定，组合是对齐列表（按字典序排序）：

特征形状是否默认用途
amplitude(280, 3, 3)✅ 必选主信号，子载波振幅
phase(280, 3, 3)opt-in净化后的相位（去 STO/CFO），M2.5 ablation
rssi(5, 3, 3)opt-in包级 RSSI（dB），用作 ×10^(RSSI/20) 振幅重缩放

开启全部特征时输入通道数翻倍到 560 = 280(amp) + 280(phase)，RSSI 不进通道数（仅作振幅 rescale）。

5.3 相位净化（phase sanitization）

每包 STO（采样时间偏移）和 CFO（中心频率偏移）会让相位出现沿子载波的线性斜坡。 用最小二乘投影去掉：

φ̃ = P · unwrap(φ),    P = I_56 − A(AᵀA)⁻¹Aᵀ,    A = [k 1] ∈ ℝ^(56×2)

P 把向量投影到 span(k, 1) 的正交补——任何斜率+常数都被剔除，剩下纯净的多径相位。 实现：host/csi_pipe/align.py:sanitized_phase。

5.4 归一化流水线

按训练集统计计算 μ、σ（验证集用同一份，不重算）：

  1. 缺损填 0 → L2（per-packet per-link）：
    X̂^(i,j) = X^(i,j) / ‖X^(i,j)‖₂
    缺损向量（≥ 2 个缺损包）填 0，归一化仍为 0（避免除零陷阱）。
  2. RSSI 重缩放（opt-in）：
    X̂^(i,j) ← X̂^(i,j) · 10^(RSSI_ij / 20)
    补偿每块板的 AGC 差异，让绝对功率可比较。
  3. Z-score（per-link，用 train 统计）：
    X = (X̂ − μ) / σ，其中 μ, σ ∈ ℝ^(3×3)
    9 个链路各自独立标准化。μ、σ 写入 checkpoint。

归一化核心代码（train/csi_train/data.py）

def normalize_rows(rows, features, mu, sigma, mu_phase=None, sigma_phase=None):
    feats = _canon_features(features)
    Xn = l2_normalize(rows.X)              # ① per-link L2
    if "rssi" in feats:
        Xn = rssi_rescale(Xn, rows.RS)     # ② RSSI 重缩放（opt）
    X = apply_stats(Xn, mu, sigma)         # ③ z-score → f16
    if "phase" in feats:
        X = np.concatenate([X, apply_stats(rows.XP.astype(np.float32),
                                           mu_phase, sigma_phase)], axis=1)
    return X
⚠️ 不跨链路用相位：6 块板子的晶振独立，跨链路相位差是随机量， 把它当特征会让模型过拟合到某次会话的随机相位偏差。只能链路内相位形状（净化后）作为 ablation。

6.1 来源与定位

WiSPPN-ESP 是 WiSPPN（Intel 5300 单 NIC，antenna matrix）的非官方移植。 来源：geekfeiw/WiSPPN， 论文 Can WiFi Estimate Person Pose?（arXiv:1904.00277）。

主要修改点（4 个）：

  1. stem 280→150 通道正式使用（原版有 conv1 未调用的 bug）
  2. decode 最后一层 2ch → 3ch（加 ĉ 通道）
  3. interpolate scale_factor=48 改为显式 size=(144,144)（等价但更明确）
  4. 新增 vector_head 变体（直接回归 18 关节向量，跳过 PAM 张量）

def forward(self, x):
    size = self.n_joints * 8                     # 144 = 18 × 8
    x = F.interpolate(x, size=(size, size),
                      mode="bilinear", align_corners=False)
    x = self.stem(x)                             # (B,150,144,144)
    x = self.layer4(self.layer3(self.layer2(self.layer1(x))))
    if self.vector_head:
        return self.head(x.mean(dim=(2, 3))).view(-1, 3, self.n_joints)
    return self.decode(x)                        # (B, 3, 18, 18)

train/csi_train/model.py · decode head（论文风格）

self.decode = nn.Sequential(
    conv3x3(width * 2, 64),    # 300 → 64
    nn.BatchNorm2d(64),
    nn.ReLU(inplace=True),
    nn.Conv2d(64, 3, 1, bias=False))    # 64 → 3 (x, y, ĉ)
💡 vector head vs PAM head：PAM 张量 (3,18,18) ≈ 972 个输出值，参数多但表达力强（理论上可输出关节对关系）； vector head (3,18) ≈ 54 个输出值，参数少训练快，但被锁定只能 diag_only 损失模式（loss.py 显式校验）。 实际部署推荐 vector head（速度、稳定性）。

6.6 训练效率细节

  • batch=64 是被内存约束定的：forward 时输入升采样到 144×144， 中间张量 (B,150,144,144) f32 单 batch ≈ 11.9 MB；batch=512 试过会触发 16 GB GPU OOM（实测）。
  • 升采样在 forward 里做而不是 dataset 里——避免在内存中物化巨型 f16 数组。
  • bf16 autocast：cuda 自动混精，CPU 强制 fp32。

7.3 损失函数

核心思路：confidence-floor weighted MSE。负样本（presence=0）不参与坐标损失，但参与 ĉ 损失（让 ĉ 学会趋零）。

w = 𝟙[presence] · max(c_gt, 0.2) ← C_FLOOR = 0.2

三种损失模式（loss.py:MODES）：

模式适用 head损失覆盖范围
pam_fullPAM head全 PAM 元素
diag_balancedPAM head对角权重 ×17（306/18 均衡）
diag_onlyvector head只对角 18 个关节

损失核心（train/csi_train/loss.py）

def pose_loss(pred, Y, presence, *, mode="pam_full", lam=1.0):
    d = torch.arange(J, device=Y.device)
    C = Y[:, 2]                              # (B, J, J) confidence
    c_diag = C[:, d, d]                      # (B, J) 对角
    pr = presence.to(Y.dtype)

    if pred.dim() == 3:                      # vector head
        w = pr.view(-1, 1) * c_diag.clamp(min=C_FLOOR)
        se = (pred[:, :2] - Y[:, :2, d, d]) ** 2
        coord = (w.unsqueeze(1) ** 2 * se).mean()
        conf  = (pred[:, 2] - c_diag).pow(2).mean()
    else:                                    # PAM head
        w = pr.view(-1, 1, 1) * C.clamp(min=C_FLOOR)
        eye = torch.eye(J, device=Y.device, dtype=Y.dtype)
        m = torch.ones_like(eye)             # pam_full
        se = (pred[:, :2] - Y[:, :2]) ** 2
        coord = (m * w.unsqueeze(1) ** 2 * se).mean()
        conf  = (pred[:, 2, d, d] - c_diag).pow(2).mean()

    total = coord + lam * conf
    return total, {"coord": coord.detach(), "conf": conf.detach()}
💡 为什么 confidence floor = 0.2？RTMPose 给的 c 值在 0.0-1.0 之间。 但低 c 帧（关节被遮挡、模糊）仍含部分有效信息——直接用 c=0 屏蔽会丢掉这些。 设下限 0.2 相当于"轻信但不全信"，对难样本友好。

7.4 数据增强（4 种）

仅训练时启用，z-score 空间操作，概率 p=0.5 per-sample：

增强参数模拟的现实扰动
Gaussian noiseσ = 0.05接收机热噪声
Scale jitterU(1±0.25)AGC 调整、距离变化
Subcarrier dropk = 4窄带衰落（一个 4-SC 簇被选择性衰减）
Link erasingp = 0.1单链路失效（板子卡顿）
💡 填充值 = mu0：drop/erasing 的填值是 -(mu / sigma)——也就是 raw 0 在 z-score 空间的值。 这样训练时缺失的子载波/链路分布与真实捕获场景一致，避免人为引入"好填充"的虚假信号。

7.5 评估指标 PCK@α

Percentage of Correct Keypoints——关节位置预测在 GT 一定比例（α）的归一化距离以内就算对。 关键创新：躺姿鲁棒分母 D_f。

D_f = 𝟙[AR < 0.8] · max(torso_f, κ · diag_f) + 𝟙[AR ≥ 0.8] · torso_f

AR = h/w（核心 bbox 高宽比，h/w < 0.8 = 躺姿），torso = 躯干对角线（肩-髋）， diag = 核心 bbox 对角线。躺下时躯干被压缩，必须用 bbox 对角线校正，否则 PCK 会虚高。

κ = median_{AR ≥ 1.2}(torso / diag)——用训练集的直立帧做一次标定，把两个量纲统一成同尺度。

7.6 Best checkpoint 选择

按 val PCK@0.2 最大 选。每个 epoch 跑一次 val，保存最优 best.pt。 checkpoint 内容：state_dict + μ + σ + config + git_rev + κ + best 元数据 + (opt) μ_phase, σ_phase。

8实时推理 + 跌倒 FSM ★（深）

8.1 推理入口

rt/demo.py 启动后选择 ReplaySource（回放 HDF5）或 LiveSource（订阅 MQTT）， 喂入 PoseEstimator（rt/csi_rt/infer.py），得到每窗口的 18 关节坐标 + 置信度， 再送进 FallDetector（rt/csi_rt/fall.py）做状态机。

rt/csi_rt/infer.py · 核心推理

def __call__(self, X_raw):
    """X_raw (280,3,3) f32/f16 → (xy (18,2) f32, c (18,) f32)"""
    Xn = l2_normalize(np.asarray(X_raw, np.float32)[None])  # ① per-link L2
    Xn = (Xn - self.mu) / self.sigma                       # ② z-score（ckpt μ·σ）
    with torch.no_grad():
        out = self.model(torch.from_numpy(Xn.astype(np.float32)).to(self.device))
    o = out[0].cpu().numpy()
    if self.vector_head:
        v = o                                              # (3, 18)
    else:
        d = np.arange(o.shape[-1])
        v = o[:, d, d]                                     # (3, 18, 18) → (3, 18) 对角
    return v[:2].T.astype(np.float32).copy(), v[2].astype(np.float32).copy()

8.5 ALARM 保持窗口

进入 IMPACT 后，不能立即报警——必须等待确认窗口内"躺且静"：

参数值含义
confirm_s2.0 sIMPACT 后等待确认的时间
lying_ratio0.7窗口内"躺姿"帧比例下界
theta_still0.02姿态帧间位移 RMS 上界（归一化空间）
refractory_s10.0 sALARM 后抑制期
release_hold_s1.0 s恢复站立需持续时间
absent_release_s5.0 s离开检测区这么久才解除
grace_s1.5 sIMPACT 后 ĉ 短暂塌陷的容忍窗口
release_hip_y0.6恢复站立时 hip_y 上限
💡 为什么要有 hold 窗口？蹲下、靠墙坐下、蹲下系鞋带这些动作都可能瞬间触发 R1+R2。 必须等 2 s 看他是不是真的躺下不动了，才能确认是跌倒。 没有这个窗口，蹲下也会被报成跌倒。

8.6 当前禁用项

⚠️ CSI 静止检测已关闭：theta_csi_still = 0。 站、躺、空房三种状态的运动能量分布严重重叠—— 实测 m15-cap1：empty p95 = 11.8 / lie p95 = 12.0 / stand p5 = 8.8（包级平均振幅 std）。 AGC 和 chunk 噪声主导了统计量，不能用来分类。 目前只能靠姿态几何判别静止，无法靠 CSI 能量。

A.3 术语表

CSI（Channel State Information）
信道状态信息，OFDM 接收机对每个子载波的复数信道估计。csi-pose 用 56 子载波。
PAM（Pose Appearance Map）
把 18 关节坐标展开成 (3, 18, 18) 张量——对角是 (x, y, ĉ)，非对角是关节对亲和度（项目未实际使用）。
PCK（Percentage of Correct Keypoints）
关节点估计正确率。距离阈用分母（躺姿鲁棒）归一化。
CFO / STO
载波频偏 / 采样时间偏。每包相位线性斜坡的两个来源，相位净化要消掉。
AGC（自动增益控制）
接收机的可变增益，会让每包振幅绝对值漂移。用 per-link L2 归一化吸收。
FSM（Finite State Machine）
有限状态机。跌倒检测用 IDLE → IMPACT → ALARM 三态。
FSM / R1/R2/R3
跌倒检测的三条规则：髋斜率 / 高宽比反转 / 鼻位置。
κ（kappa）
躺姿 PCK 分母标定系数，train 集直立帧 median(torso/diag)。
C_FLOOR
confidence 下限 0.2，避免低 c 帧被完全屏蔽。
